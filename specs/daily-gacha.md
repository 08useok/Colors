# 일일 가샤 (랜덤 뽑기) — 베타 시즌 1

## 개요
매일 최대 6회까지 가샤(랜덤 뽑기)를 돌릴 수 있다. 가샤는 코인/크레딧/꾸미기 아이템을 확률적으로 지급하는 유일한 방식이며, **크레딧은 오직 이 가샤를 통해서만 획득 가능**하다 (상점 구매·전투 보상 등 다른 경로 없음).

## 횟수 규칙

- 하루 최대 6회
- `daily-login.md`의 날짜 로직(`getTodayString()`)과 동일한 기준으로 자정 기준 리셋
- 계정 스키마 추가:
  ```json
  "gacha": { "lastResetDate": "2026-07-06", "remaining": 6 }
  ```
- 날짜가 바뀌면 `remaining`을 6으로 리셋 (로그인 시 `daily-login`과 동일한 타이밍에 체크)
- `remaining`이 0이면 가샤 버튼 비활성화, "내일 다시 시도하세요" 안내

## 보상 테이블 (가중치 기반)

아래 가중치는 상대값이며, 실제 당첨 확률은 `가중치 / 전체 가중치 합`으로 정규화한다.

### 코인
| 보상 | 가중치 |
|---|---|
| 50코인 | 1/3 |
| 100코인 | 1/6 |
| 200코인 | 1/9 |
| 400코인 | 1/18 |

### 크레딧
| 보상 | 가중치 |
|---|---|
| 50크레딧 | 1/4 |
| 100크레딧 | 1/8 |
| 200크레딧 | 1/16 |
| 400크레딧 | 1/32 |

### 꾸미기 아이템
| 보상 | 가중치 |
|---|---|
| 이모트 1종 (5종 중 1개, 각각) | 1/7 |
| 뱃지 1종 (`badge_none` 제외 7종 중 1개, 각각) | 1/16 |

- 프로필 배경(bg)은 가샤 풀에서 **제외** — 배경은 기존처럼 코인으로만 구매 가능
- `badge_none`("없음")은 이미 전원 기본 보유 상태이므로 가샤 풀에서 제외

## 중복 당첨 처리

- 이모트/뱃지가 당첨되었는데 이미 보유 중이면, 해당 아이템의 상점 코인 가격만큼 코인으로 자동 환산하여 지급한다
  - 예: 이미 보유한 `emote_fire`(300코인) 당첨 → 코인 300 지급
  - 예: 이미 보유한 `badge_gem`(300코인) 당첨 → 코인 300 지급
- 결과 화면에는 "이미 보유 중 → 코인으로 전환" 문구와 함께 실제 지급된 코인량 표시

## 결과 UI

- 가샤 버튼 클릭 시 1회 소모, 결과를 팝업/토스트로 표시 (아이템 아이콘 + 이름 + 수량)
- 잔여 횟수를 가샤 버튼 근처에 "오늘 남은 횟수: N/6" 형태로 표시
- 신규 이모트/뱃지 당첨 시 `ownedEmotes`/`ownedBadges`에 즉시 추가 (자동 장착은 하지 않음 — 기존 장착 유지)

## Non-goals
- 가샤 확률 UI 공개(확률표를 게임 내에 노출하는 기능) — 이번 시즌 범위 밖
- 가샤 횟수 추가 구매(코인/크레딧으로 횟수 충전)
- 캐릭터/스킨 자체를 가샤로 직접 지급 (캐릭터·스킨은 상점에서 크레딧/코인으로 구매하는 기존 경로 유지)
- 배경(bg) 아이템의 가샤 지급

## Acceptance Criteria
- [ ] 하루 최대 6회까지 가샤를 돌릴 수 있다
- [ ] 자정이 지나면(날짜 변경) 잔여 횟수가 6으로 리셋된다
- [ ] 잔여 횟수가 0이면 가샤 버튼이 비활성화된다
- [ ] 코인/크레딧/이모트/뱃지가 지정된 가중치 비율로 지급된다
- [ ] 배경(bg)과 `badge_none`은 가샤 풀에 포함되지 않는다
- [ ] 이미 보유한 이모트/뱃지가 당첨되면 해당 아이템의 코인 가격만큼 코인으로 자동 환산되어 지급된다
- [ ] 가샤 결과가 화면에 즉시 표시된다
- [ ] 신규 이모트/뱃지 당첨 시 보유 목록에 즉시 추가된다
- [ ] 지급된 코인/크레딧은 계정에 즉시 반영되어 잔액이 갱신된다
