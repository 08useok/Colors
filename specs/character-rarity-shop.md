# 캐릭터 등급 · 크레딧 구매 시스템

## 개요
캐릭터별로 등급(일반/희귀/영웅)을 부여하고, 희귀·영웅 등급은 신규 화폐 **크레딧**으로 구매해야 사용할 수 있도록 한다. 크레딧은 오직 `specs/daily-gacha.md`의 일일 가샤로만 획득 가능하다.

## 캐릭터 등급표

| 등급 | 캐릭터 | 가격 |
|---|---|---|
| 일반 (무료) | Red, Green, Blue | 무료 (기본 보유) |
| 희귀 | Orange, Yellow, Cyan, Purple, Pink | 각 200크레딧 |
| 영웅 | Crimson | 900크레딧 |

## 계정 스키마 추가

```json
"ownedCharacters": ["red", "green", "blue"],
"credits": 0
```

- `ownedCharacters`: 보유(잠금 해제)한 캐릭터 id 배열
- `credits`: 보유 크레딧 잔액

## 기존 계정 마이그레이션 (중요)

베타 시즌 1 이전에 생성된 기존 계정은 등급 시스템 도입 이전부터 8캐릭터(Red/Green/Blue/Orange/Yellow/Cyan/Purple/Pink)를 모두 자유롭게 사용해왔다. 이 유저들에게 불이익이 없도록:

- `account.ownedCharacters`가 없는 기존 계정을 감지하면, **8캐릭터 전부**를 `ownedCharacters`에 자동으로 채워 넣는다 (`["red","green","blue","orange","yellow","cyan","purple","pink"]`)
- `credits` 필드가 없으면 0으로 초기화
- 신규 캐릭터 크림슨은 기존 계정도 예외 없이 **미보유 상태로 시작** (자동 언락 대상 아님 — 신규 콘텐츠이므로 900크레딧 구매 필요)
- 신규 계정(베타 시즌 1 이후 생성)은 `ownedCharacters: ["red", "green", "blue"]`로 시작

## 캐릭터 선택 화면 잠금 표시

- `ownedCharacters`에 없는 캐릭터 버튼은 잠금 아이콘(🔒) + 등급별 가격 표시 (예: "🔒 200크레딧" / "🔒 900크레딧")
- 잠긴 캐릭터 버튼 클릭 시 선택되지 않고, 대신 "상점에서 구매" 안내 또는 상점 캐릭터 구매 탭으로 이동
- 보유한 캐릭터만 정상적으로 선택 가능

## 상점 구매 플로우

- 기존 상점(`#shop-overlay`)에 캐릭터 구매 UI 추가 (레벨업 탭 또는 신규 탭 — 미보유 캐릭터만 노출)
- 구매 버튼 클릭 → 크레딧 잔액 확인
  - 부족: 토스트 "크레딧이 부족합니다" 표시
  - 충분: 크레딧 차감 → `ownedCharacters`에 id 추가 → 즉시 캐릭터 선택 화면에서 잠금 해제
- 구매 후 `account` localStorage 저장

## AI 봇 배정
- 봇 캐릭터 랜덤 배정(`botTypes` 등)은 유저의 보유 여부와 무관하게 기존처럼 9종(크림슨 포함 시 9종) 전체 중 랜덤 — 등급/보유 제한은 플레이어 전용

## Non-goals
- 크레딧의 코인 환전/코인의 크레딧 환전 (양방향 환전 없음)
- 크레딧 실제 결제(IAP) 연동 — 오직 가샤로만 획득
- 캐릭터 등급 재분류(향후 시즌에 등급 변경) — 이번 시즌 값만 정의

## Acceptance Criteria
- [ ] 신규 계정은 Red/Green/Blue만 보유한 상태로 시작한다
- [ ] 기존 계정(베타 이전 생성)은 접속 시 8캐릭터가 자동으로 `ownedCharacters`에 채워진다
- [ ] 기존 계정도 크림슨은 미보유 상태로 시작한다
- [ ] 미보유 캐릭터는 선택 화면에서 잠금 아이콘과 가격이 표시되고 선택되지 않는다
- [ ] 상점에서 크레딧으로 희귀/영웅 캐릭터를 구매하면 즉시 보유·선택 가능해진다
- [ ] 크레딧이 부족하면 구매가 진행되지 않고 안내 메시지가 표시된다
- [ ] 구매 후 크레딧 잔액이 즉시 차감되어 표시된다
- [ ] AI 봇은 플레이어의 캐릭터 보유 여부와 무관하게 전체 캐릭터 풀에서 랜덤 배정된다
