# 배틀 맵 로테이션 시스템

## 개요
"전투 시작" 시 여러 배틀 맵 중 하나를 랜덤으로 선택해 콘텐츠 다양성을 높이고, 반복 플레이의 단조로움을 줄이며, 캐릭터별 사거리/밸런스를 다양한 지형에서 테스트할 수 있게 한다. 훈련장은 영향받지 않는다.

## 맵 풀 구성
| 맵 | 설명 |
|---|---|
| Map A (기존) | 현재 배틀 맵의 벽/호수/수풀/스컬 배치 그대로 |
| Map B (신규) | 새로운 벽/호수/수풀/스컬 배치 |
| Map C (신규) | 새로운 벽/호수/수풀/스컬 배치 |

각 맵은 다음 항목을 독립적으로 정의한다:
- 맵 이름 (HUD 표시용, 예: "해골 협곡", "마른 호수", "...")
- 벽 배치 (`wallSpecs` 형태)
- 수풀 배치 (`bushSpecs` 형태)
- 스컬 클러스터 배치 (`skullSpecs` 형태)
- 호수 위치/크기 (없을 수도 있음)
- 플레이어 스폰 포인트 10개 (벽·호수와 겹치지 않는 위치, `worldRadius` 내부)

모든 맵에서 동일하게 유지되는 공유 요소:
- `worldRadius = 52`
- 자기장(Zone) 5단계 축소 로직 및 초기 `safeCenter = (0, 0)`
- 캐릭터 스탯 및 무기 밸런스 (6캐릭터 공통)

## 로테이션 동작
1. 로비에서 "전투 시작" 클릭 → 맵 풀(Map A/B/C) 중 1개를 랜덤 선택 → 선택된 맵 ID를 상태에 저장
2. 선택된 맵의 데이터로 배틀 맵을 구성 (`battleSolids`, `battleLakeRects`, `battleBushes`, 배틀 맵 그룹의 3D 객체)
3. 직전 전투에서 사용한 맵의 3D 객체·`solids`·`lakeRects`·`bushes`는 잔여물 없이 제거 후 새 맵으로 교체
4. "다시 시작"(Play Again) 클릭 → 새로 랜덤 선택하지 않고, 직전 전투와 동일한 맵을 재사용
5. "훈련장" 버튼 → 로테이션과 무관하게 기존 고정 훈련 맵을 그대로 사용

## HUD 표시
- 전투 중 좌상단 패널(top-left)에 현재 맵 이름을 표시한다 (예: "맵: 해골 협곡")
- 훈련장에서는 맵 이름을 표시하지 않는다 (기존 UI 유지)

## Non-goals
- 맵 미리보기 화면 / 맵 투표 UI
- 맵별 자기장 크기·축소 단계·캐릭터 밸런스 차등
- 훈련장 맵 로테이션 (훈련장은 항상 고정)
- 투사체의 벽 충돌 (기존 비목표 유지)

## Acceptance Criteria
- [ ] 맵 풀에 총 3개의 맵(기존 1개 + 신규 2개)이 존재한다
- [ ] "전투 시작" 클릭 시 3개 맵 중 하나가 랜덤으로 선택되어 적용된다
- [ ] 선택된 맵에 따라 벽/호수/수풀/스컬/스폰포인트가 올바르게 적용된다
- [ ] 맵 전환 시 이전 맵의 3D 객체·solids·lakeRects·bushes·effects·projectiles가 잔여물 없이 정리된다
- [ ] "다시 시작" 클릭 시 직전 전투와 동일한 맵이 유지된다 (재추첨 없음)
- [ ] 전투 중 HUD 좌상단에 현재 맵 이름이 표시된다
- [ ] 훈련장은 항상 기존 고정 맵을 사용한다 (로테이션 미적용, 맵 이름 표시 없음)
- [ ] 신규 맵 2개 모두에서 AI 봇 10명이 정상적으로 스폰·이동·전투한다 (스폰 포인트가 벽/호수와 겹치지 않음)
- [ ] 신규 맵 2개 모두에서 자기장 5단계 축소가 기존과 동일하게 동작한다
